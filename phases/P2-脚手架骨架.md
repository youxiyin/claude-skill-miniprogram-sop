# P2 · 脚手架 + 骨架（喂给 agent 的指令）

把 P1 确认的 HTML 转成**能跑、门禁全绿、循环即插即用**的真小程序骨架。**全自动，人闸③默认略过。**

## 你要做的
### A. HTML → 真小程序
1. 每个确认页 → wxml/wxss/js，页面结构与视觉忠于 `prototype/` 基线。
2. 建目录：`app.js`（globalData + 数据层入口）、`app.json`（页面注册、tabBar、`__usePrivacyCheck__`）、`utils/`（数据层 data-store：本地持久化 + 云同步骨架）、`cloudfunctions/`（每个云函数**从第一天就带 `config.json`**：openapi 权限 / 定时触发器 / env 占位——**绝不留部署缺口**）。
3. 组件先出**空壳但可渲染**（有空态、不白屏），逻辑留给循环填。

### B. 接入循环基建（从老项目 `automation/` 复制骨架，改路径/云函数名）
- `loop-state.json`（字段见 SOP §2，初值清零，cadence 数值填好，`risk_register`/`spec_coverage`/`journey_coverage` 指向 A 生成的资产）；
- `round-prompt.md`（13 步 + **12 条铁律**，领域红线一句指向 `risk-register.md`）；
- `supervisor.sh` / `watchdog.sh` / `enable.sh` / `disable.sh` / `notify-*.sh` / `decision-digest.sh`；
- **门禁审计脚本**（进 `npm test`）：`deploy-config-audit`（权限/触发器）、`behavior-lock-spec-citation-audit`（锁测试引出处）、`realdevice-checklist`（改动页→手测清单）、`e2e-flow`（连不上优雅跳过）、`fixture-authenticity-audit`（外部输入解析器 ≥1 真实样本，铁律#12）。

### C. 视觉保真锁（防"确认时好看、实现后走样"）
- 截实现页面图，与 `prototype/` 的 HTML 基线比对；
- 把关键视觉差值（间距/字号/主色/CTA 层级/触控盒大小）**降维成可断言的差值**，写一条视觉保真回归测试，锁住"用户确认的样子"。

### D. 冒烟验收
- `npm test` 必须**全绿**；
- 先跑 `enable.sh` 起 **1–2 轮 discovery**，验"资产真能抓 bug"（首跑常当场抓出部署即触发的真隐患=资产有效信号）。

## 交接闸（人闸③，默认略过）
骨架期不值得用户深看；不立住 → 你自愈重来。仅当用户主动要看时才递骨架预览。

## 铁律（本阶段相关）
- 门禁全绿才算完成，红的不许提交；每修一条单独绿灯提交。
- 云函数**同步维护 config.json**（部署配置完整性，铁律#9）。
- 跨端复制的逻辑（算钱/排程/日期）**必配漂移测试**（铁律#8）。
- 外部输入解析器（OCR/识别）**必须有真实样本语料**，否则视同未验证（铁律#12）——骨架期若还没真实样本，登记为 `fixture_debt` 催用户补，别用合成占位冒充。
