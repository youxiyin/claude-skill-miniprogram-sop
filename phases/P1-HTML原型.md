# P1 · HTML 原型（喂给 agent 的指令）

从 P0 的 spec/页面清单，出**静态 HTML 原型**让用户看效果、确认方向。**这是全流程最关键的一道确认，也是最便宜的修改时机。**

## 你要做的
1. **每个页面出一版 HTML**，放 `prototype/<页面名>.html`，统一套 `prototype-shell/_mobile-frame.html` 的手机边框 + 设计令牌（见该目录 README）。
2. **关键状态各出一版**：空态 / 正常 / 满载 / 异常（如加载失败、无网络、无权限）。适老化类还要出**大字号版**。
3. **页面间可点击跳转**：底 tab、列表点进详情、返回——让用户能"走一遍"，不是看孤立截图。
4. **保真约束（重要）**：只用**能干净映射到 wxml 组件**的结构（view/text/image/scroll-view/swiper/button/input…）。**不要**用 wxml 做不出的花活（复杂 CSS 动画、canvas 特效、grid 高级布局），否则 P2 转换必走样。
5. **推给用户看**：能浏览器直接打开；若经飞书/微信桥，把关键页截图或文件推过去。

## 交接闸（人闸②·核心闸）
- 用户看效果 → 指出"哪页不对 / 少了什么 / 这感觉不对"。
- 你改，**反复到用户说"就是这个感觉"**才放行。
- 放行 = 这套**页面结构 + 视觉基调冻结成基线**：
  - 把定稿 HTML 保留在 `prototype/`（P2 要拿它做视觉保真回归的比对基准）；
  - 记一句 `prototype/BASELINE.md`：冻结时间 + 页面清单 + "此后循环不许擅自漂移视觉基调，除非用户再发话"。

## 别做
- 别在这一步接数据/写逻辑/连云——**原型只管"长什么样、怎么走"**。
- 别自作主张加/删页面偏离 spec；要加先回 P0 更新 spec 再出图。
- 方向没确认前，**别碰真小程序代码**（P2 才动）。

## 为什么是 HTML 不是真小程序
HTML 改一版是秒级、零构建、零开发者工具、零部署。方向/信息架构/视觉基调这类**最不可逆**的决策，在这里修最便宜；等进了真小程序再改方向，成本翻几十倍。
