# 电商/批发类红线（种子来自批发电商项目实战）

**总红线**：**钱、货、单**三样不能错。凡涉及算钱、算量、改单状态、支付，错了直接亏钱或纠纷 → 严重度默认 **S3/P0**。

| 编号 | 失效模式 | 严重度 | 触发条件 | 守护方向（锁测试） |
|---|---|---|---|---|
| E1 | **金额错算**：客户端算的和服务端算的不一致 | S3 | 同一算价逻辑两端各写一份、悄悄分叉 | **漂移测试**（两实现同批输入结果一致，铁律#8）+ 服务端为准 |
| E2 | **超卖/批次超发**：库存扣减错、并发下超卖 | S3 | 库存判断非原子/客户端说了算 | 库存扣减原子性 + 边界锁测试 |
| E3 | **订单状态 key 误改**：状态流转写错 key，订单卡死或跳错态 | P0 | 状态机 key 硬编码/拼错 | 订单状态机锁测试 |
| E4 | **支付误触/重复扣款**：按钮误触、无幂等导致重复支付 | S3 | 缺二次确认/缺幂等 key | 幂等锁测试 + 支付前确认闸 |
| E5 | **留资闭环漏**：用户留了联系方式/需求但没进后台，商机丢 | P1 | 提交后无落库/无回执 | 端到端旅程回归（留资→落库→通知）|
| E6 | **营销规则泄漏/被薅**：优惠叠加、越权领券、规则写在客户端 | P1 | 规则可被客户端绕过 | 规则服务端校验 + 边界测试 |
| E7 | **溯源错配**：批次/产地/证书对错货 | P1 | 溯源数据关联错 | 关联一致性锁测试 |
| E8 | **上云串档**：多端/账号数据同步串到别人 | P0 | 身份守卫缺失/LWW 冲突 | 身份守卫 + 稳定 id 合并（G2）|

> **P0 必问用户**：是否涉及真实支付结算？（决定 E1/E4 的守护强度）
