# 医疗健康类红线（种子来自「健康管家」实战）

**总红线**：帮家庭/患者**不漏事**，但**不替医生下判断**。凡涉及吃药、复查、指标判读、提醒送达，错了会直接害到人 → 严重度默认 **S3（凌驾一切调度，未护视同 P0）**。

| 编号 | 失效模式 | 严重度 | 触发条件 | 守护方向（锁测试） |
|---|---|---|---|---|
| M1 | **漏服**：该提醒的没提醒、排程算错少发 | S3 | 频率排程/日期计算错、定时触发器没配 | 排程属性测试 + 部署触发器审计 + 送达回执 |
| M2 | **过量**：重复提醒、算多了、误判"还没吃" | S3 | 打卡状态覆盖、去重失效 | 打卡稳定 id + 去重锁测试（漏发↔误发双向）|
| M3 | **误判已服**：把没吃标成吃了 | S3 | 状态写错/多端覆盖 | 状态机锁测试 |
| M4 | **提醒未送达**：云函数没部署/无权限/退化成 dry-run | S3 | 缺 config.json（触发器+openapi 权限）| deploy-config 审计 + 真机订阅回执 |
| M5 | **识别漏指标**：OCR/报告解析漏掉真指标 | S3 | 合成语料照绿、真实报告结构没覆盖 | 真实语料金标门禁 + 蜕变 + 语料真实性护栏（铁律#12）|
| M6 | **危急值漏判/误判**：超标不报 或 正常误报 | S3 | 判读逻辑单向收紧 | 双向断言（漏判↔误判）+ 参考区间属性测试 |
| M7 | **替医生下结论**：给出诊断/用药建议越过"帮记录"边界 | S3 | 文案/功能越界 | 意图对抗 lens 核边界，越界即 finding |
| M8 | **健康数据串档**：空 openid/身份混淆导致 A 的报告进 B 名下 | S3 | 云函数身份守卫缺失 | 空身份拦截锁测试（getOpenId 守卫）|
| M9 | **真实输入未验**：识别类能力只在编的数据上验过 | P0 | 金标语料只有合成占位 | fixture-authenticity 门禁 + 催人补真实脱敏样本 |

> **P0 必问用户**：这个小程序会不会替医生做判断？（决定 M7 的边界怎么划）
