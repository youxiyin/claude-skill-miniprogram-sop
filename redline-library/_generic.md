# 通用红线（任何小程序都套，_generic）

P0 起草 `risk-register.md` 时，先把下面这些**无条件纳入**（除非明显不适用），状态一律 ❌未护，交给循环去护。

| 编号 | 失效模式（什么会出错·怎么害用户） | 严重度 | 典型触发 | 守护方向（锁测试） |
|---|---|---|---|---|
| G1 | **核心数据丢失**：没有兜底持久化，退出/崩溃后数据没了 | P0 | 只存内存/未落盘/未同步云 | 持久化+恢复锁测试 |
| G2 | **多端覆盖（LWW 冲突）**：多设备/云回载后互相覆盖，新数据被旧数据顶掉 | P1 | 无时间戳/无稳定 id 的 last-write-wins | 每条记录带稳定 id + 录入时刻戳，合并语义锁测试 |
| G3 | **边界输入崩溃**：空/超长/特殊字符/异常格式让页面白屏或崩 | P1 | 未校验的输入直接用 | 空态+异常态锁测试、组件注册审计 |
| G4 | **部署/基建缺口**：调云能力没声明权限、定时任务没触发器、密钥没配 | P0 | 单测全是桩，看不见部署缺口 | deploy-config 审计进门禁 |
| G5 | **导航意图错配**：点这个字/图标，落点不是用户想去的地方 | P1 | 测试锁的是"代码碰巧行为" | fresh-eyes 意图对抗 lens + E2E 落点断言 |
| G6 | **缺失功能/负空间**：该有的入口没有、或有入口走不到落点 | P1 | 读代码只看见"在的" | 规格→UI 可达性清单 |
| G7 | **隐私越权**：收集/使用了没声明的权限，或用途与声明不符 | P0 | app.json 未开隐私校验、用途栏空 | 隐私指引逐项 + `__usePrivacyCheck__` |
| G8 | **加载/失败无反馈**：请求中/失败时无提示，用户以为卡死或重复操作 | P2 | 缺 loading/error 态 | 状态真实性锁测试 |

> **怎么用**：这 8 条是地板。品类文件在此之上加领域红线。做完一个新项目，把新踩的通用坑补回本文件。
