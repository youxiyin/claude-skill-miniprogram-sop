# miniprogram-sop · Claude Code Skill

从**一段话**到**可上线**的微信小程序标准化制作流水线，做成一个可 `/` 调用的 [Claude Code](https://claude.com/claude-code) skill。

> 一句话：你给一段话 → agent 先出 **HTML 原型**让你确认方向 → 确认后脚手架成真小程序 → 交给**自动迭代循环**长跑 → 只把不可替代的判断打包递给你，其余全自动。

## 它做什么：5 阶段 + 5 道人闸

```
一段话 → P0 需求澄清 → [人闸①澄清] → P1 HTML原型 → [人闸②方向确认·核心]
       → P2 脚手架骨架 → [人闸③略过] → P3 自动迭代循环 → [人闸④产品取舍]
       → P4 真机·部署·上线 → [人闸⑤真机+提审] → 可上线
```

**只有 5 类事必须人来**：① 方向对不对 ② 产品/业务取舍 ③ 真机验证 ④ 真实数据/语料 ⑤ 部署/平台提审。其余每个阶段由 agent 自动做到"人闸前一步"，并把闸门做成一键判断。

## 安装

克隆到你的 Claude Code 个人技能目录（仓库根即技能本体）：

```bash
git clone <this-repo> ~/.claude/skills/miniprogram-sop
```

下次会话即可用 `/miniprogram-sop <你的小程序想法>` 触发；或直接说"帮我做个 XX 小程序 / 先出个 HTML 原型看看"。

## 结构

```
miniprogram-sop/
├── SKILL.md              ← 编排器（入口）：驱动 5 阶段 + 5 人闸 + 12 铁律
├── SOP-full.md           ← 完整 SOP 正文（设计理念、诚实人闸清单，按需查）
├── phases/               ← P0/P1/P2 三份"喂给 agent 的阶段指令"
├── templates/            ← spec-checklist / risk-register / user-journeys 空模板
├── redline-library/      ← 品类红线库（通用+医疗+电商有实战内容；社交/工具/预约待沉淀）
└── prototype-shell/      ← HTML 原型外壳（手机边框+设计令牌，只用可映射 wxml 的结构）
```

## 设计原则
1. 把每类缺陷推到**最便宜的阶段**修：方向→HTML、逻辑数据→循环、真机部署→上线。
2. **领域知识 agent 起草、人只审批**（不让人填空）。
3. **人闸=判断不是劳动**：选项/默认/清单都备好，人只点选/点头/拍照。

## 复利
做完一个新品类项目，把新踩的红线补回 `redline-library/<品类>.md`——做的项目越多，P0 起草越准。
