# 小程序制作标准化 SOP v1
### —— 从「一段话」到「可上线」，除人必须判断处外，每一阶段自动做到位

> **一句话**：你给一段话描述 → 我先出 **HTML 原型**让你确认方向 → 确认后自动脚手架成真小程序 → 交给**自动迭代循环**长期打磨 → 最终把只有你能拍板的少数几件事**打包成一键判断**递给你，其余全自动做好。
>
> 本 SOP 是外层「生产流水线」，把已成熟的 [通用自动迭代循环设计 v1](通用小程序自动迭代循环设计-v1.md)（那套 L5 循环）当作其中**一个阶段**的引擎。循环负责"把一个已存在的项目越打磨越好"，本 SOP 负责"从零把项目立起来、并在每个阶段前一步把决策递给人"。

---

## 0. 三条设计原则（为什么这样排）

1. **把每类缺陷推到它最便宜的阶段去修。**
   - 方向/信息架构错 → 最贵（推翻重来）→ 放到 **HTML 原型阶段**，秒级迭代、零构建。
   - 逻辑/数据/边界错 → 中等 → 放到**自动迭代循环阶段**，门禁+对抗校验兜住。
   - 真机/部署/平台错 → 只有真环境暴露 → 放到**上线阶段**，结构化清单+人闸。
   > 你"先看 HTML 效果再做"的直觉正是这条：**把最不可逆的决策（方向），用最便宜的介质（HTML）最早确认。**

2. **领域知识由 agent 起草、人只审批。**
   - 老循环把「风险登记册/规格清单/用户旅程」当作**你手写**的输入。本 SOP 让 agent 从你那段话 + **品类红线模板库**自动起草，你在确认闸**只做判断题**（对/改/补），不做填空题。这才是"除人必须判断处外都做好"。

3. **人闸 = 判断，不是劳动。** 每道人闸，agent 都已把选项、建议默认、对比材料、点击清单备齐，你的动作压缩成"选一个 / 点个头 / 拍一张"。**凡能被 agent 做的都做完，只把不可替代的判断留给你。**

## 2. 阶段详解

### P0 · 需求澄清（agent 全自动，产出领域知识草稿）
- **输入**：你的一段话。
- **agent 做什么**：
  1. 判品类（医疗/电商/社交/工具/内容/预约…），套**对应红线模板库**（§4）。
  2. 抽结构化规格：核心用户、3–5 条核心闭环、页面清单、每页要素、数据实体。
  3. 起草三份**领域知识资产**（老循环里"唯一要你手写"的东西，现在自动生成）：
     - `spec-checklist.md`（功能→端到端可达且符合意图）
     - `risk-register.md`（失效模式目录，含品类模板红线 + 从描述推断的红线）
     - `user-journeys.md`（核心闭环 J1…Jn，每步"应发生什么"）
  4. 只在**关键信息缺失且无法安全默认**时，生成 **2–3 个澄清问题**（如：医疗类必问"是否替医生判断"；电商必问"金额是否涉支付"）。
- **产出**：上述四份草稿 + （可选）澄清问题。
- **人闸①**：仅当有澄清问题时你回答；否则 agent 带**标注了假设的默认**直接进 P1（假设写进 spec，后续可推翻）。

### P1 · HTML 原型（★核心：你看效果确认方向）
- **agent 做什么**：把全部页面生成**静态 HTML**——手机边框、贴近最终风格的样式、页面间可点击跳转、关键状态（空/满/异常）各出一版。用可直接浏览器打开 / 推你飞书或微信预览。
- **为什么是 HTML 不是真小程序**：HTML 改一版是**秒级**、零构建、零开发者工具、零部署。方向类缺陷（信息架构、页面流、视觉基调、有没有漏页面）在这里修**最便宜**。等进了真小程序再改方向，成本翻几十倍。
- **保真约束**：HTML 只用**能干净映射到 wxml 组件**的结构（避免用 wxml 做不出的花活），保证 P2 转换不走样。
- **人闸②（方向确认，全流程最关键的一道）**：
  - 你看效果 → 说"哪页不对/少了什么/这个感觉不对"。
  - agent 改，反复到你说**"就是这个感觉"**才放行。
  - 放行 = 这套**页面结构与视觉基调被冻结成基线**，后续循环不许擅自漂移（除非你再发话）。

### P2 · 脚手架 + 骨架（agent 全自动）
- **agent 做什么**：
  1. 确认的 HTML → 真小程序页面（wxml/wxss/js）、目录结构、数据层（本地+云同步骨架）、云函数骨架（含 config.json/权限/触发器占位——**从第一天就不留部署缺口**）。
  2. 接入**循环基建**（直接复制老循环骨架）：`loop-state.json`（初值清零、cadence 填好）、`round-prompt.md`（13 步 + §3 铁律）、`supervisor/watchdog/enable/disable/notify/decision-digest`、门禁审计脚本（部署审计/锁测试引用/真机清单/E2E/**语料真实性**）。
  3. **视觉保真回归**：截实现页面的图，与 P1 确认的 HTML 原型比对，锁住"确认的样子"——防"确认时好看、实现后走样"。
- **产出**：一个能跑起来、门禁全绿、循环即插即用的**骨架小程序**。
- **人闸③（可选，轻量）**：跑一遍，扫一眼骨架立住没（不立住 agent 自愈重来）。骨架期不需你深判，故此闸默认略过。

### P3 · 自动迭代循环（= 通用循环 v1 全套，agent 长期无人值守）
- **就是那套 L5 循环**：多角度只读子 agent 发现 → 对抗校验（防注水）→ 逐条修 → 锁测试（防回归、防骗绿）→ 门禁全绿单条提交 → judge 对抗判分 → 分层收敛。
- **模式机**：discovery（找新 bug）→ polish（燃尽 backlog+视觉+规格）→ converging（只收尾签字）。
- **调度**：cadence 保底 + bandit 往高产 lens 加码 + MAP-Elites 保广度 + 双车道（发现/燃料）。
- **铁律**（在老循环 10 条上补齐健康管家最新两条，共 12 条，见 §3）。
- **人闸④**：循环遇到**只有你能拍板**的（产品取舍/主观 UI/文案）→ **不自己拍**，追加进 `human_decisions_needed`，攒够或到 SLA 时**打包**推你，低风险项**预填建议默认**（你不反对下轮自动采纳），高风险项（医疗红线/钱/数据/平台）**强阻塞**等你。**决策只进能出**：被代码现实推翻的自动撤下。

### P4 · 真机 · 部署 · 上线（agent 备清单，你执行不可替代动作）
- **agent 做什么**：按本批改动**自动生成**——真机手测清单（改了哪页→该测什么流程→具体点哪几下）、云函数部署清单（哪些函数改了要重部署、哪些 env 要在控制台配）、平台提审材料（隐私保护指引逐项 + 内容安全接入核对）。
- **人闸⑤（不可消除）**：真机点一遍、云函数部署、平台提审——**机器碰不到真手机、不能 push/deploy、不能替你在微信后台提交**。这是物理边界，不是能力不足。agent 的职责是把这一公里**看得见、清单化、每轮怼到你面前**，绝不假装"我这边绿=可上线"。

## 3. 铁律（12 条，绝对遵守）

前 10 条见 [通用循环 v1 §4]；健康管家实战又补两条，**新项目直接带上**：

11. **双向目标，禁单向收紧（防越修越坏）**：每条堵漏侧的护栏收紧，**必须成对**配一条"别误伤"的反向断言（漏发↔误发、少识别↔幻象、拦截↔误拦）。凡碰关键计算（算钱/排程/库存/依从/日期/识别/闸门）除点测试外**必扩一条蜕变关系或属性不变量**，且该模块变异分不许降。**护栏要建模"正确行为"，不是记住某个 bug 的形状**（否则换个形状就绕过、护栏库只单调膨胀却悄悄压掉真阳性）。
12. **真实输入语料护栏（防"合成占位照绿"）**：凡解析**真实世界外部输入**的模块（OCR/证件/语音/图片识别），其"已验证"资格**受语料出处封顶**——金标语料若**只有 agent 编的合成占位**，该能力**视同未验证**，门禁绿也不许收敛。硬锁"每个此类解析器 ≥1 份真实样本 + 每份声明出处"；真实格式多样性不足 = 常驻债，每轮催人补样本（**机器造不出真实数据，只能守住已有的、把缺口路由给人**）。触碰此类解析器时须从真实样本**派生结构变体**（真凶多是"结构轴缺失"而非某个值错）。

## 4. 品类红线模板库（P0 自动套用，本 SOP 的"复用资产"）

老循环说风险登记册"唯一要你手写"。本 SOP 把它升级为**按品类预置的模板库**，agent 判出品类即套用、再按描述增删，你只在人闸①审。种子来自实战两例，逐步扩充：

| 品类 | 预置红线（失效模式）示例 |
|---|---|
| **医疗/健康** | 漏服/过量/误判已服/提醒未送达/识别漏指标/危急值漏判或误判/替医生下结论/数据串档 |
| **电商/批发** | 金额错算(端↔云不一致)/超卖/订单状态 key 误改/支付误触/留资闭环漏/营销规则泄漏/溯源错配/上云串档 |
| **社交/内容** | 内容安全未过审/隐私越权/未成年保护/举报闭环漏/消息丢失或串人 |
| **工具/效率** | 数据丢失(无兜底持久化)/多端覆盖(LWW 冲突)/导出错数据/边界输入崩溃 |
| **预约/服务** | 时段超订/时区错算/通知未达/改约退约状态不一致/双重扣款 |

> 每个新品类跑完一次，就把新发现的红线沉淀回模板库——**做的项目越多，P0 起草越准**。这是这套 SOP 的复利。

## 5. 诚实清单：到底哪些"人必须判断"，哪些"全自动"

你的目标是"除人必须判断处外都做好每一阶段"。把这条**兑现成一张诚实的表**——不夸大 agent 能力，也不把能自动的甩给你：

**★ 只有人能做的（不可消除，共 5 类）**
1. **方向对不对**（人闸②）——你脑子里的"感觉"，机器造不出，只能出图让你选。
2. **产品/业务取舍**（人闸④）——如"归档报告能不能改指标"这类有真实权衡、无客观对错的题。
3. **真机验证**（人闸⑤）——机器碰不到真手机、真微信、真订阅推送。
4. **真实数据/语料**（贯穿，铁律#12）——真实报告/证件/录音，机器造不出，只能守住你给的、催你补缺的格式。
5. **部署/平台动作**（人闸⑤）——push/deploy/微信后台提审/密钥，机器不越权。

**✓ agent 全自动做好的（其余全部）**
- 需求抽取、领域知识起草、HTML 出图、脚手架、真小程序实现、视觉保真锁定；
- 找 bug、对抗校验防注水、修复、锁测试防回归、防"把 bug 当正确"、收敛判定；
- 部署配置完整性审计、跨端逻辑漂移守卫、缺失功能/负空间排查、端到端旅程回归；
- 债计龄催办、决策打包预填、真机/部署/提审清单生成、每轮推送摘要、崩溃自愈、限额续跑、静默死亡看门狗。

> 一句话给你兜底预期：**这套 SOP 让 agent 把每个阶段自动推进到"人闸前一步"，并把闸门做成一键判断。它不能替你上线，但能让"到上线之间的每一步"都不用你操心，且从不拿"我这边绿"糊弄你。**

## 6. 每个新项目的启动清单（照此 30–60 分钟起跑）

1. **发一段话** → agent 跑 P0，产出 spec/风险/旅程草稿 + 澄清问题。
2. **答澄清（若有）** → 人闸①。
3. **看 HTML 原型** → 反复到"就是这个感觉" → 人闸②冻结基线。
4. **agent 跑 P2** → 脚手架+骨架+循环基建+视觉保真锁，门禁全绿。
5. **`enable.sh` 起循环**，先跑 1–2 轮 discovery **验资产真能抓 bug**（实战首跑常当场抓出部署即触发的真隐患=资产有效信号），再放手长跑。
6. **循环长跑**，你只在飞书里应人闸④（打包决策，多是点选）。
7. **到里程碑** → agent 出真机/部署/提审清单 → 你走人闸⑤ → 上线。

```
